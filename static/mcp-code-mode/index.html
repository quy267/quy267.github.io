Triển khai an toàn

Lăn bánh an toàn: read-only trước, rồi mới mở đường write.

1. Read-only — agent query & chẩn đoán (đơn lỗi, giao dịch stuck)
2. Write có kiểm soát — "7 câu hỏi": ai gọi? idempotent? read/write? có log/trace?
3. Self-host — nếu vướng data residency; pattern portable khi bạn kiểm soát runtime/dữ liệu

Bề mặt cho agent là quyết định thiết kế — đừng để mọi endpoint mở mặc định.

▸ Beat 12 · ~1.5 phút. Read-only là cửa an toàn nhất để bắt đầu · 7 câu hỏi = cổng trước khi MCP-ify · then →
